# UI Patterns

Patterns are common, repeatable UI solutions that combine multiple atomic components to solve a specific user problem.

## Common Pattern Categories

### 1. Forms
Guidelines for collecting user input.
- **Included Components:** Inputs, selects, checkboxes, radios, buttons.
- **Considerations:** Input grouping, validation messaging (inline vs. summary), submission states, required field indicators.

### 2. Navigation
How users move through the application.
- **Included Components:** Sidebars, top nav bars, tabs, breadcrumbs, pagination.
- **Considerations:** Information architecture, active state indicators, responsive behavior (e.g., hamburger menus).

### 3. Data Display
How to present information clearly.
- **Included Components:** Data tables, cards, lists, descriptions.
- **Considerations:** Sorting, filtering, pagination, empty states, responsive layouts (e.g., tables converting to cards on mobile).

### 4. Feedback
How the system communicates status to the user.
- **Included Components:** Toasts/Snackbars, Modals/Dialogs, inline alert messages, progress indicators.
- **Considerations:** Interruptive vs. non-interruptive feedback, auto-dismissal rules, clear calls to action within the feedback.
