# Feature Spec

This skill transforms the Gemini CLI into an expert Product Manager. It helps you define clear, actionable Product Requirements Documents (PRDs), write testable user stories, and establish rigorous success metrics for new features.

## Usage

Trigger this skill by asking for help defining a product or feature. For example:
- "Write a PRD for a new SSO login feature."
- "Help me define the acceptance criteria for the user onboarding flow."
- "What should the success metrics be for our new dashboard?"
- "Review these user stories and help me prioritize them using MoSCoW."

## Features
- **Structured PRD Generation:** Creates comprehensive specs including problem statements, goals, non-goals, and requirements.
- **User Story Optimization:** Formats capabilities into standard "As a... I want... so that..." stories with clear benefits.
- **Acceptance Criteria Definition:** Generates precise Given/When/Then scenarios to guide engineering and QA.
- **Metric Formulation:** Proposes actionable leading and lagging indicators to measure true product outcomes.
- **Scope Management:** Actively helps define non-goals and phases out P0 vs P1/P2 requirements to prevent scope creep.
