# UX Copy Patterns

## 1. Calls to Action (CTAs)
- **Rule:** Start with a strong, descriptive verb. Match the label to the exact outcome.
- **Good:** "Create account", "Save changes", "Download PDF"
- **Bad:** "Submit", "Click here", "OK"

## 2. Error Messages
- **Structure:** [What happened] + [Why it happened] + [How to fix it]
- **Rule:** Never blame the user. Be specific and actionable.
- **Example:** "Payment declined. The expiration date is invalid. Please update your card details or use a different payment method."

## 3. Empty States
- **Structure:** [What this space is for] + [Why it's empty] + [Clear CTA to start]
- **Rule:** Treat empty states as onboarding opportunities. Encourage action.
- **Example:** "No projects yet. This is where your team's work will live. Create your first project to get started."

## 4. Confirmation Dialogs
- **Structure:** [Clear Headline] + [Consequence Description] + [Action Buttons]
- **Rule:** The primary button should reiterate the action, not just say "Yes" or "OK".
- **Example:**
  - Headline: "Delete 3 files?"
  - Body: "These files will be permanently removed from your account. This action cannot be undone."
  - Buttons: "Cancel" (Secondary) / "Delete files" (Primary/Destructive)

## 5. Success Messages
- **Rule:** Confirm the action was successful briefly. Offer the next logical step if applicable.
- **Example:** "Profile updated successfully. View your public profile."
